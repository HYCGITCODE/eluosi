## 设计概述

### 设计理念
- **简洁高效**: 最小化认知负担，核心功能一目了然
- **直观交互**: 拖拽、双击等自然手势操作
- **视觉清晰**: 优先级颜色编码，状态明确区分
- **响应迅速**: 实时反馈，加载状态友好

### 目标用户
- 需要管理日常任务的个人用户
- 偏好视觉化时间管理的用户
- 需要快速记录和调整任务的用户

## 视觉规范

### 颜色系统

| 用途 | 色值 | 说明 |
|------|------|------|
| **P0 优先级** | `#EF4444` (红) | 高优先级，紧急重要 |
| **P1 优先级** | `#F59E0B` (黄) | 中优先级，重要不紧急 |
| **P2 优先级** | `#10B981` (绿) | 低优先级，日常任务 |
| **主色调** | `#3B82F6` (蓝) | 品牌色，按钮、链接 |
| **背景色** | `#FFFFFF` (白) | 主背景 |
| **次要背景** | `#F9FAFB` (浅灰) | 卡片、对话框背景 |
| **边框色** | `#E5E7EB` (灰) | 分隔线、边框 |
| **文字主色** | `#111827` (深灰) | 主要文字 |
| **文字次要** | `#6B7280` (中灰) | 次要文字、提示 |
| **成功色** | `#10B981` (绿) | 完成状态、成功提示 |
| **警告色** | `#F59E0B` (黄) | 提醒、警告 |
| **危险色** | `#EF4444` (红) | 删除、错误 |

### 优先级颜色应用

```css
/* 任务卡片背景色 (浅色版本) */

.priority-p1 { background-color: #FEF3C7; border-left: 4px solid #F59E0B; }
.priority-p2 { background-color: #D1FAE5; border-left: 4px solid #10B981; }

/* 任务卡片文字色 (深色版本) */
.priority-p0-text { color: #EF4444; }
.priority-p1-text { color: #F59E0B; }
.priority-p2-text { color: #10B981; }

/* 完成状态 */
.completed { 
  text-decoration: line-through; 
  opacity: 0.6;
  background-color: #F3F4F6;
}
```

## 交互流程

### 1. 创建任务流程

```
用户操作                    系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
1. 双击日历格子    →   弹出创建对话框          →   模态框淡入
                                                背景变暗
2. 输入任务标题    →   实时验证 (必填)         →   空状态提示
3. 选择日期时间    →   默认填充双击日期        →   日期高亮
4. 选择优先级      →   即时预览颜色            →   卡片颜色变化
5. 设置重复规则    →   显示下次执行时间        →   提示文本更新
6. 设置提醒        →   计算提醒时间            →   显示提醒时间
7. 点击保存        →   验证 → 保存 → 关闭     →   成功 Toast
                                                新任务卡片出现
```

**详细步骤**:

1. **触发动作**: 用户双击日历格子
2. **对话框打开**: 
   - 模态框从中心淡入 (300ms)
   - 背景添加半透明遮罩 (rgba(0,0,0,0.5))
   - 焦点自动定位到标题输入框
3. **表单填写**:
   - 标题：必填，实时验证，空时显示"请输入任务名称"
   - 日期：默认双击日期，可修改
   - 时间：可选，默认 09:00
   - 优先级：默认 P1，单选按钮
   - 重复：默认"不重复"，下拉选项 (每天/每周/每月/每年)
   - 提醒：默认关闭，开启后显示时间选择 (5/10/15/30 分钟前)
   - 描述：可选，多行文本
4. **保存操作**:
   - 点击"保存"按钮
   - 前端验证 (标题必填)
   - 显示加载状态 (按钮变 loading)
   - 保存到 localStorage/数据库
   - 关闭对话框 (淡出 200ms)
   - 显示 Toast 提示："任务已创建" (2 秒后自动消失)
   - 新任务卡片出现在对应日期格子

### 2. 编辑任务流程

```
用户操作                    系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
```

2. **对话框打开**:
   - 模态框淡入
   - 表单预填充当前任务数据
   - 焦点定位到标题输入框
3. **修改内容**: 同创建流程
4. **保存操作**:
   - 验证 → 保存 → 关闭

### 3. 删除任务流程

```
用户操作                    系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
1. 点击删除按钮    →   弹出确认对话框          →   模态框淡入
                                                警告图标

                                                成功 Toast
```

2. **确认对话框**:
   - 显示任务名称

3. **确认删除**:
   - 点击"删除"按钮
   - 从存储中删除
   - 卡片淡出 + 缩小动画 (300ms)
   - Toast 提示:"任务已删除"
4. **取消删除**:
   - 点击"取消"或关闭按钮

### 4. 拖拽修改日期流程

```
用户操作                    系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
1. 按住任务卡片    →   进入拖拽状态            →   卡片半透明
                                                悬浮阴影增大
2. 拖动到目标日期  →   显示目标日期高亮        →   目标格子背景色
                                                插入位置指示线
3. 释放鼠标        →   更新任务日期 → 保存    →   卡片移动动画
                                                成功 Toast
4. 拖动中 ESC      →   取消拖拽                →   卡片回到原位
```

**详细步骤**:

1. **开始拖拽**:
   - 鼠标按下任务卡片
   - 延迟 200ms 进入拖拽状态 (区分点击和拖拽)
   - 卡片变为半透明 (opacity: 0.7)
   - 创建拖拽镜像 (跟随鼠标)
2. **拖拽过程**:
   - 镜像跟随鼠标移动
   - 经过的日期格子高亮 (background: #EFF6FF)
   - 显示插入位置指示线 (蓝色，4px)
   - 实时更新目标日期提示
3. **释放拖拽**:
   - 在有效目标格子上释放
   - 更新任务日期字段
   - 保存到存储
   - 原位置卡片淡出
   - 新位置卡片淡入
   - Toast 提示:"任务已移动到 X 月 X 日"
4. **取消拖拽**:
   - 按 ESC 键
   - 或在无效区域释放
   - 卡片回到原位置
   - 无数据变更

### 5. 切换完成状态流程

```
用户操作                    系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
1. 点击复选框      →   切换完成状态            →   复选框勾选
```

**详细步骤**:

2. **样式更新**:
   - 添加 `.completed` 类
   - 文字添加删除线

   - 背景色变灰 (#F3F4F6)
3. **数据保存**:
   - 更新任务的 completed 字段
   - 保存到 localStorage/数据库
   - 显示短暂 Toast:"已标记完成"(1 秒)
4. **取消完成**:
   - 再次点击复选框
   - 移除 `.completed` 类
   - 恢复原样式

### 6. 搜索任务流程

```
用户操作                    系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
1. 点击搜索框      →   获得焦点                  →   边框高亮
                                                显示搜索图标
2. 输入搜索词      →   实时过滤 (防抖 300ms)    →   匹配结果高亮
                                                非匹配项变淡
3. 查看结果        →   高亮显示匹配任务          →   黄色背景高亮
4. 清除搜索        →   恢复全部显示              →   清空输入框
                                                全部任务显示
```

**详细步骤**:

1. **激活搜索**:
   - 点击搜索框或按 Ctrl+F
   - 搜索框获得焦点
   - 边框变蓝 (#3B82F6)
2. **输入搜索**:
   - 实时监听输入 (防抖 300ms)
   - 搜索范围：任务标题、描述
   - 匹配算法：包含匹配 (不区分大小写)
3. **显示结果**:
   - 匹配任务：黄色背景高亮 (#FEF3C7)
   - 非匹配任务：透明度降低 (opacity: 0.3)
   - 无匹配：显示"未找到匹配任务"
   - 自动滚动到第一个匹配项
4. **清除搜索**:
   - 点击清除按钮 (×)
   - 或清空输入框
   - 所有任务恢复正常显示
   - 搜索框失去焦点

### 7. 过滤任务流程

```
用户操作                    系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
1. 点击过滤按钮    →   展开下拉菜单              →   下拉菜单淡入
```

   - 下拉菜单保持打开 (可多选)
3. **应用过滤**:
   - 前端过滤显示的任务
   - 非匹配任务隐藏 (display: none)
   - 显示过滤标签："P0 · 未完成"

   - 点击"清除"按钮

   - 移除过滤标签

```
用户操作                    系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
```

2. **视图过渡**:

   - 新视图淡入 (opacity: 0→1, 200ms)

   - 月视图：7 列×5-6 行网格
   - 周视图：时间轴 +7 列，显示具体时间段

### 9. 提醒触发流程

```
时间到达      →   系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
提醒时间到    →   检查任务状态                →   弹窗出现
未完成       →   显示提醒弹窗                →   应用图标闪烁
                                                系统通知
标记完成     →   关闭弹窗 → 更新状态          →   弹窗消失
稍后提醒     →   设置新提醒时间                →   弹窗消失
                                                显示下次提醒时间
```

**详细步骤**:

1. **定时检查**:
   - 每 60 秒检查一次提醒队列
   - 对比当前时间与任务提醒时间
2. **触发提醒**:
   - 任务未完成且到达提醒时间
   - 显示提醒弹窗 (右上角)
   - 播放提示音 (可选)
   - 发送系统通知 (Notification API)
   - 应用图标闪烁 (favicon)
3. **弹窗内容**:
   - 任务名称
   - 开始时间
   - 优先级标识
   - 两个按钮:"稍后提醒"(5/10/15 分钟), "标记完成"
4. **用户操作**:
   - 标记完成：更新任务状态，关闭弹窗
   - 稍后提醒：设置新提醒时间，关闭弹窗
   - 关闭弹窗：暂时关闭，下次仍会提醒

### 10. 数据持久化流程

```
用户操作                    系统响应                    视觉反馈
─────────────────────────────────────────────────────────────────
创建/修改任务  →   保存到 localStorage        →   自动保存提示
页面刷新      →   从存储加载数据              →   加载状态

```

**详细步骤**:

1. **自动保存**:
   - 每次创建/修改/删除任务后

2. **数据结构**:

   ```json
     ],
     "settings": {
       "view": "month",
       "filters": {
         "priority": "all",
         "status": "incomplete"
       }
     }
   }
   ```
3. **加载数据**:
   - 应用启动时读取 localStorage
   - 显示加载动画 (如数据量大)
   - 恢复视图状态、过滤条件
   - 渲染任务列表
4. **数据备份** (可选):
   - 导出功能：下载 JSON 文件
   - 导入功能：上传 JSON 文件
   - 云同步：未来扩展

## 性能优化

### 渲染优化

- 虚拟滚动 (长列表)
- 懒加载 (图片、图表)
- 防抖 (搜索、拖拽)
- 节流 (滚动、窗口调整)

### 数据优化

- 增量更新 (非全量渲染)
- 本地缓存 (localStorage)
- 按需加载 (周/日视图数据)

### 动画优化

- CSS 动画优先 (GPU 加速)
- 减少重排重绘
- 动画时长 <300ms
