## 五、核心问题总结

### 5.1 沟通闭环缺失 (高优先级)
- **问题**: QA 通知延迟 50 分钟，因无正确联系人 ID
- **根因**: 依赖单次消息发送，无"发送 - 确认 - 执行"闭环
- **影响**: QA 测试延迟，整体进度受影响
- **改进**: 建立通知确认机制，重要通知必须"收到回复"

### 5.2 流程执行松懈 (中优先级)
- **问题**: PM 中断 5 小时，TEAM.md 章程执行不够严格
- **根因**: 在"效率"和"流程"之间选择了前者
- **影响**: 项目进度停滞，测试阶段压缩
- **改进**: 建立"风险 15 分钟上报"机制，超时自动升级

### 5.3 部署验证不足 (中优先级)
- **问题**: Railway 部署配置未经验证，导致部署失败
- **根因**: 部署配置只写文件，未实际测试
- **影响**: 切换至本地部署，增加工作量
- **改进**: 部署配置必须经过实际验证，提供自动化脚本

### 5.4 前后端接口对齐不足 (低优先级)
- **问题**: 字段命名和响应格式不一致，导致 2 次返工
- **根因**: API 设计未与前端提前对齐
- **影响**: 测试阶段发现，需紧急修复
- **改进**: 建立 API 评审机制，FE/BE 共同确认接口规范
